    - None -> ("OK", "OK") — D-10: daily_sales=0 case

    **aggregate_sales_in_range(monthly_sales: dict[str, int], start_date: date, end_date: date) -> int:**
    - Suma ventas entre start_date (inclusive) y end_date (inclusive)
    - Usa prorrateado proporcional para meses parciales: units * (days_covered / days_in_month)
    - days_covered = (overlap_end - overlap_start).days + 1  (CRITICAL: +1 para inclusión de ambos extremos — Pitfall 1)
    - usa calendar.monthrange para días del mes (maneja años bisiestos)
    - Retorna int(total_float)
    - Itera desde primer día del mes de start_date hasta end_date, avanza mes a mes
    - Implementación exacta según Pattern 2 del RESEARCH.md

    **calculate_in_transit_real(sku: str, pending_orders_for_sku: list[dict], monthly_sales: dict[str, int], today: date) -> int:**
    - pending_orders_for_sku: lista de dicts con {"quantity": int, "created_at": datetime (timezone-aware)}
    - D-03: si lista vacía -> return 0
    - D-01: sum(o["quantity"] for o in pending_orders_for_sku) -> total_pending_qty
    - D-01: oldest_date = min(o["created_at"].date() for o in pending_orders_for_sku)
    - D-02: sales_since_oldest = aggregate_sales_in_range(monthly_sales, oldest_date, today)
    - D-01: absorbed = min(sales_since_oldest, total_pending_qty)
    - D-01: return max(0, total_pending_qty - absorbed)
    - GUARD: created_at puede ser string ISO 8601 o datetime; normalizar con datetime.fromisoformat() si str

    **calculate_replenishment(inventory_items: list[dict], sales_cache: dict[str, dict[str, int]], pending_orders_map: dict[str, list[dict]], params: dict) -> dict:**
    - params keys: lead_time_days (int), safety_factor (float), date_range_days (int), date_range_start (date), date_range_end (date)
    - D-15: incluir TODOS los productos del inventory (incluyendo zero-velocity)
    - D-16: el filtro de vendors ya fue aplicado antes de llamar (en el endpoint)
    - Para cada item en inventory_items:
      - sku = item["sku"], stock = item["available"], title = item["title"], vendor = item["vendor"]
      - monthly_sales = sales_cache.get(sku, {})
      - total_sold = aggregate_sales_in_range(monthly_sales, date_range_start, date_range_end)
      - days_in_period = (date_range_end - date_range_start).days + 1
      - daily_sales = total_sold / days_in_period
      - sales_per_month = total_sold / max(1, (days_in_period / 30))
      - sales_per_week = daily_sales * 7
      - days_of_inventory = (stock / daily_sales) if daily_sales > 0 else None  (Pitfall 2: division guard)
      - reorder_point = daily_sales * params["lead_time_days"] * params["safety_factor"]
      - needs_reorder = stock <= reorder_point
      - in_transit = pending_orders_map.get(sku, [])
      - in_transit_real = calculate_in_transit_real(sku, in_transit, monthly_sales, date_range_end)
      - suggested_qty_raw = (daily_sales * params["lead_time_days"] * params["safety_factor"]) - stock - in_transit_real
      - suggested_qty = max(0, math.ceil(suggested_qty_raw))   (D-05)
      - classification, classification_label = classify_product(sales_per_month)
      - urgency, urgency_label = classify_urgency(days_of_inventory)
      - D-08: si suggested_qty == 0 -> el producto IGUAL aparece en results con urgency="OK"
    - Construir lista `products` (lista de dicts, un dict por item)
    - Ordenar `products` por days_of_inventory ascending (None al final, usar 9999 como proxy)

    - D-17, D-18: Agregar por vendor:
      - Para cada vendor único en products:
        - total_skus = count de products con ese vendor Y suggested_qty > 0
        - total_units_to_order = sum de suggested_qty para ese vendor
        - urgent_count = count de products con urgency == "URGENTE" Y suggested_qty > 0
      - Ordenar vendor_summary: primero por urgent_count DESC, luego por total_units_to_order DESC (D-18)
      - Solo incluir vendors donde total_units_to_order > 0
