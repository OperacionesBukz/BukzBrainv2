  <action>
Add a new rule block for the `notifications` collection in `firestore.rules`, placing it after the existing `agent_conversations` block and before the closing braces.

Add this rule block:
```
    // --- Notifications ---
    // Users can read their own notifications and update them (mark as read).
    // Any authenticated user can create notifications (trigger functions create for others).
    match /notifications/{notifId} {
      allow read: if isAuthenticated()
                  && resource.data.userId == request.auth.token.email;

      allow create: if isAuthenticated();

      allow update: if isAuthenticated()
                    && resource.data.userId == request.auth.token.email;

      allow delete: if isAdmin();
    }
```

IMPORTANT: The user has a rule in CLAUDE.md "NO modificar firestore.rules sin confirmacion explicita del usuario". However, this is part of the planned notification system that the user requested. Add the rules but include a comment at the top of the new block: `// Added for notification system (Phase 1)`.

Note: `createNotificationForAdmins` creates docs with `userId` set to admin emails, so admins can read their own notifications. The `create` rule is open to authenticated users because any user action (like creating a leave request) may trigger a notification for admins.
