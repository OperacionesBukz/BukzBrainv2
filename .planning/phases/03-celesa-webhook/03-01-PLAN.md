# Plan 03-01: Webhook Shopify Orders/Fulfilled → Celesa Auto-Entregado

## Overview
Crear endpoint webhook que recibe notificaciones de Shopify cuando un pedido se marca como Fulfilled, busca pedidos Celesa con el mismo Order Name en Firestore, y actualiza su estado a "Entregado".

## Tasks

### Task 1: Agregar SHOPIFY_WEBHOOK_SECRET a config.py
**File:** `backend/config.py`
**Action:** Agregar `SHOPIFY_WEBHOOK_SECRET` al class Settings
**Details:**
- `SHOPIFY_WEBHOOK_SECRET: str = os.getenv("SHOPIFY_WEBHOOK_SECRET", "")`

### Task 2: Crear router webhooks.py
**File:** `backend/routers/webhooks.py` (NEW)
**Action:** Crear router con verificacion HMAC y endpoint orders/fulfilled
**Details:**
- `APIRouter(prefix="/api/webhooks", tags=["Webhooks"])`
- Funcion `verify_hmac(body: bytes, hmac_header: str) -> bool` con hmac.compare_digest
- Endpoint `POST /shopify/orders-fulfilled`:
  1. Leer body raw (antes de parse JSON)
  2. Verificar HMAC con X-Shopify-Hmac-Sha256 header
  3. Parsear payload JSON
  4. Extraer `name` del payload (es el order_name, ej: "#192197")
  5. Query Firestore: `celesa_orders` where `numeroPedido == order_name`
  6. Para cada match: si estado no es "Entregado" ni "Agotado", actualizar a "Entregado" + updatedAt
  7. Retornar 200 OK siempre (Shopify requiere 200 para no reintentar)
- Health check endpoint

### Task 3: Registrar router en main.py
**File:** `backend/main.py`
**Action:** Import y registrar webhooks router SIN auth dependency
**Details:**
- Import: `from routers import webhooks`
- Register: `app.include_router(webhooks.router)` (sin `dependencies=_auth`)

## Verification

- [ ] Pedidos en estado "Entregado" o "Agotado" no se sobrescriben
