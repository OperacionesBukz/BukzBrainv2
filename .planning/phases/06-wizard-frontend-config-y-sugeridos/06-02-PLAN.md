Add a `useCalculationFlow` custom hook to the existing hooks.ts file. This hook encapsulates the multi-step async calculation flow (per D-04, D-05, D-06) so that index.tsx stays clean.

**Hook signature:**
```typescript
interface CalculationFlowState {
  isPolling: boolean;
  isCalculating: boolean;
  salesStatus: SalesStatusResponse | null;
  results: CalculateResponse | null;
  error: string | null;
}

export function useCalculationFlow(): CalculationFlowState & {
  startCalculation: (request: CalculateRequest) => Promise<void>;
  resetResults: () => void;
}
```

**Internal implementation:**
1. State: `isPolling: boolean`, `pendingRequest: CalculateRequest | null`
2. Uses `useSalesStatusPolling(isPolling)` internally for polling
3. Uses `useCalculate()` mutation internally

**`startCalculation(request)` flow:**
1. Check sales cache status: call `getSalesStatus()` from api.ts directly (import from `./api`)
2. If `status === "completed"` and `last_refreshed` is < 24h ago: proceed directly to `calculateMutation.mutate(request)` (per D-05)
3. If cache stale/missing: call `refreshSales(request.date_range_days)`, set `isPolling = true`, store `pendingRequest = request`, show toast "Actualizando datos de ventas..." (per D-04)
4. If 409 error (OPERATION_IN_PROGRESS): show `toast.error("Hay una operacion Bulk en curso en Shopify. Intenta en unos minutos.")` (per D-06). Do NOT set polling.
5. `useEffect` watching polling data: when `salesStatus?.status === "completed"` and `pendingRequest` exists: set `isPolling = false`, trigger `calculateMutation.mutate(pendingRequest)`, clear `pendingRequest`
6. On calculate success: set `results`, show toast "Calculo completado"
7. On calculate error: show `toast.error(error.message)`

**Cache freshness check:** Compare `last_refreshed` ISO string to `Date.now()`. Fresh if less than `24 * 60 * 60 * 1000` ms difference.

Import `toast` from `sonner`, `getSalesStatus` and `refreshSales` from `./api`.
